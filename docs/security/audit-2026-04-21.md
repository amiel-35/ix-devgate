# Audit de Sécurité DevGate — 2026-04-21

**Périmètre :** Backend FastAPI · Frontend Next.js · Dépendances · Secret store  
**Findings :** 2 critiques · 5 élevés · 8 moyens · 6 faibles

---

## Résumé exécutif

L'architecture de sécurité de DevGate est globalement solide pour une v1 : secret store AES-256-GCM correct, séparation session/grant dans le gateway, anti-enumeration email, single-use tokens, audit systématique. Deux failles de logique d'autorisation critiques doivent être corrigées avant tout déploiement en production.

## CRITIQUE

### C-01 — Logout ne révoque pas la session en base
**`apps/api/app/modules/auth/router.py:48`**

`POST /auth/logout` supprime le cookie mais ne supprime pas la session en base. Un cookie capturé reste valide 7 jours après déconnexion.

```python
# Fix
db.delete(current_session)

response.delete_cookie(SESSION_COOKIE, path="/", samesite="lax",
                        secure=settings.COOKIE_SECURE, httponly=True)
```

## ÉLEVÉ

### H-01 — `require_agency_admin` : condition `kind="agency"` bypass le contrôle de grant
**`apps/api/app/shared/deps.py:48`**

```python
# Bug : un user kind="agency" sans grant agency_admin passe
if not has_admin_grant and user.kind != "agency":  # OR logique dangereux
```
Tout utilisateur avec `kind="agency"` accède au back-office complet sans grant admin explicite.

```python
# Fix
def require_agency_admin(user: User = Depends(get_current_user)) -> User:
    has_admin_grant = any(g.role == "agency_admin" and not g.revoked_at for g in user.grants)
    if not has_admin_grant:
        raise ForbiddenException("Réservé aux administrateurs agence")
    return user
```

### M-06 — Absence de Security Headers HTTP (backend + frontend)
**`apps/api/app/main.py` · `apps/web/next.config.ts`**

Aucun header de sécurité configuré : HSTS, CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy.

```python
# Backend (FastAPI)
class SecurityHeadersMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        response = await call_next(request)
        response.headers["X-Content-Type-Options"] = "nosniff"
        response.headers["X-Frame-Options"] = "DENY"
        response.headers["Strict-Transport-Security"] = "max-age=63072000"
        return response
```

```typescript
// Frontend (next.config.ts)
async headers() {
    return [{ source: "/(.*)", headers: [
        { key: "X-Frame-Options", value: "DENY" },

        { key: "Content-Security-Policy", value: "default-src 'self'; frame-ancestors 'none';" },
```

**`apps/api/app/modules/secrets/store.py:109`**

## Ce qui est bien implémenté ✅

- **Secret store** : AES-256-GCM, nonce aléatoire 96 bits, AAD déterministe — implémentation correcte
- **Anti-timing** : comparaison sur hash SHA-256 des tokens de challenge
- **Anti-enumeration** : réponse identique pour email connu/inconnu sur `/auth/start`
- **Single-use challenges** : `used_at` vérifié avant commit, replay impossible
- **Séparation session/grant** dans le gateway : les deux sont vérifiés avant tout proxy
- **Credentials CF jamais exposés** : `service_token_ref`, `upstream_hostname` absents des réponses API
- **`X-DevGate-User` spoofing bloqué** : header client supprimé, réinjecté côté serveur
- **Cookies `HttpOnly`** : le token de session n'est pas accessible depuis le JavaScript
- **Aucune raw SQL query** identifiée — ORM partout
